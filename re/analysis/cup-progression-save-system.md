# Cup Progression, Race Series Scheduling & Save/Restore System

Unified deep-dive covering the full lifecycle of cup races: schedule generation, per-race
configuration, mid-cup serialization, encrypted save/load, checksum validation, unlock
awards, and the "Continue Cup" flow.

## 1. Game Types and the Cup System

Test Drive 5 has 9 game types. Types 1-6 are multi-race **cup series**; types 7-9 are
single-race standalone modes.

| ID | Name | Races | Rule Variant | Difficulty | Scoring |
|----|------|-------|-------------|------------|---------|
| 1 | Championship | 4 | 0 | Easy (0) | Points: 15/12/10/5/4/3 per position |
| 2 | Era | 6 | 5 | Easy (0) | Time (lower wins), 4 laps forced |
| 3 | Challenge | 6 | 1 | Medium (1) | Time (lower wins) |
| 4 | Pitbull | 8 | 2 | Hard (2) | Time (lower wins) |
| 5 | Masters | 10 | 3 | Medium (1) | Time (lower wins) |
| 6 | Ultimate | 12 | 4 | Hard (2) | Points: 1000/500/250/0/0/0 per position |
| 7 | Time Trial | 1 | - | Hard (2) | Best time (milliseconds) |
| 8 | Cop Chase | 1 | - | - | Damage/wanted score |
| 9 | Drag Race | 1 | - | - | Time (lower wins) |

### `ConfigureGameTypeFlags` (0x410CA0)

Called before each race to configure runtime mode flags from `g_selectedGameType`:

```
Input:  g_selectedGameType (1-9)
Output: gRaceRuleVariant, gRaceDifficultyTier, gTrafficActorsEnabled,
        gSpecialEncounterEnabled, gTimeTrialModeEnabled, gWantedModeEnabled,
        gDragRaceModeEnabled, DAT_004b0fa8 (checkpoint mode), DAT_004aaf74 (HUD mode),
        gCircuitLapCount (Era only: forced to 4)
```

**Key runtime flags per type:**

| Type | Traffic | Encounters | gDragRaceModeEnabled | DAT_004aaf74 (HUD) |
|------|---------|------------|---------------------|---------------------|
| 1-6 | Yes | Yes | 0 | 2 (race HUD) |
| 7 | No | No | 0 | 3 (time trial HUD) |
| 8 | Yes | No | 0 | 4 (wanted HUD) |
| 9 | No | No | 1 | 1 (drag HUD) |

**2-player override:** If `DAT_004962a0 != 0` (2-player mode) or `g_selectedGameType == 7`
(time trial), `DAT_004aaf74` is set to 5 (split-screen HUD).

**NPC group selection for cup races:** After flag setup, the function reads from a 2D
lookup table at `0x4640A4`:

```c
DAT_00490ba8 = table[g_selectedGameType * 16 + g_raceWithinSeriesIndex + 0x14];
```

Value `0x63` (99) is the end sentinel -- when reached, the cup is finished.

**Masters mode special init (race 0 only):** Generates a 15-entry random permutation at
`DAT_0048f30c` (unique track indices 0-26), then randomly marks 5 of them as "pending"
(`DAT_0048f324[i] = 1`).

**End-of-cup dispatch:** When `DAT_00490ba8 == 99`, the function calls through a jump
table at `0x464104` indexed by game type (6 small thunks at 0x410F60..0x4110A0). These
set final state flags before returning 0 to signal cup completion.

## 2. Race Series Schedule

### Data Structure

The series state is a set of parallel arrays at `0x497250` (total 0x78 = 120 bytes):

```c
struct RaceSeriesState {
    // At 0x497250 (12 bytes padding/header)
    byte  slot_status[6];    // 0x49725C: per-race status (0=empty, 1=in-progress, 2=done)
    int   track_index[6];    // 0x497268: track pool index for each race
    int   direction[6];      // 0x497280: track direction (0-3, random)
    int   accum_time[6];     // 0x497298: accumulated time score per race
    int   accum_points[6];   // 0x4972B0: accumulated point score per race
};
```

Additional state for 2-player:
- `DAT_0049726C`: P2 track index
- `DAT_00497284`: P2 direction
- `DAT_0049729C`: P2 accumulated time
- `DAT_004972B4`: P2 accumulated points

### `InitializeRaceSeriesSchedule` (0x40DAC0)

Called when entering a cup, quick race, or returning from options screens. Guarded by
`DAT_00497324 != 0` (skip if already initialized).

**Three scheduling strategies based on game type:**

#### Strategy A: Era Cup (type 2) -- Random from Pool

```
for each of 6 race slots (with status == 0):
    pick random track index 0-7 (or 8-15 if DAT_0048f364 > 7)
    pick random direction 0-3
    ensure no duplicate track indices across the 6 slots
    set status = 2, accum_time = 0, accum_points = 0
```

The pool split at index 8 is based on `DAT_0048f364` (the first-race track from
`DAT_0048f370`), so "advanced" Era races use tracks 8-15.

#### Strategy B: Masters Cup (type 5) -- Pre-shuffled Queue

Uses the 15-entry random permutation generated by `ConfigureGameTypeFlags`:

```
for each of 5 race slots (index 1..5):
    find next entry in DAT_0048f324[] with value 1 ("pending")
    set that entry to 2 ("claimed")
    track_index = DAT_0048f30c[entry_index]  // the permuted track ID
    direction = rand() % 3
    accum_time = 0, accum_points = 0
    status = 2
```

`MarkMastersRaceSlotCompleted` (0x40DF80) marks the current track's permutation entry as
done (value 2) after each race.

#### Strategy C: All Other Cups (types 1, 3, 4, 6) -- Difficulty-Tiered Table

```
for each race slot (with status == 0):
    pick random index into difficulty_pool[gRaceDifficultyTier]
    difficulty pool at 0x463E10, 3 tiers of 6 tracks each:
      Tier 0 (Easy):   [10, 14, 7, 4, 11, 9]
      Tier 1 (Medium): [9, 6, 3, 13, 12, 0]
      Tier 2 (Hard):   [22, 24, 16, 17, 19, 18]
    ensure no duplicate track indices
    direction = rand() % 4
    accum_time = 0, accum_points = 0
    status = 2
```

### Race-to-NPC-Group Mapping Table (0x4640A4)

This 2D table (indexed as `[gameType * 16 + raceIndex]`) maps each race in a cup to one
of the 26 NPC racer groups. The offset `+0x14` in the access means the first 20 bytes
(game type 0, which is "single race") are skipped.

```
Championship (1): group 0, 1, 2, 3                              [sentinel 99]
Era          (2): group 4, 16, 6, 7, 5, 17                      [sentinel 99]
Challenge    (3): group 0, 1, 2, 3, 15, 8                       [sentinel 99]
Pitbull      (4): group 0, 1, 2, 3, 15, 8, 11, 13               [sentinel 99]
Masters      (5): group 0, 1, 2, 3, 15, 8, 11, 13, 10, 12       [sentinel 99]
Ultimate     (6): group 0, 1, 2, 3, 15, 8, 11, 13, 10, 12, 9, 14 [sentinel 99]
```

This means each progressive cup type is a strict superset of the previous: Championship
races the first 4 tracks, Era uses the 6 special lap-time tracks, Challenge adds tracks
15 and 8, and so on up to Ultimate which races all 12 unique point-to-point tracks.

## 3. Mid-Cup Snapshot Serialization

### `SerializeRaceStatusSnapshot` (0x411120)

Captures the entire cup state into a contiguous 12966-byte buffer at `0x490BAC`:

```
Offset  Size    Source                      Content
------  ------  -------------------------  --------------------------------
0x0000  1       g_selectedGameType          Game type (1-9)
0x0001  1       g_raceWithinSeriesIndex     Current race index in series
0x0002  1       DAT_004A2C90               NPC group index
0x0003  1       DAT_00490BA8               Track/opponent state
0x0004  1       gRaceRuleVariant           Rule variant (0-5)
0x0005  1       gTimeTrialModeEnabled      Time trial flag
0x0006  1       gWantedModeEnabled         Cop chase flag
0x0007  1       gRaceDifficultyTier        Difficulty (0-2)
0x0008  1       DAT_004B0FA8               Checkpoint mode
0x0009  1       gTrafficActorsEnabled      Traffic flag
0x000A  1       gSpecialEncounterEnabled   Encounter flag
0x000B  1       gCircuitLapCount           Lap count
0x000C  4       (computed CRC-32)          Checksum of entire buffer
0x0010  120     RaceSeriesState (0x497250) Schedule: tracks, dirs, scores
0x0088  120     DAT_0048D988               Race results table (6 x 20 bytes)
0x0100  12624   gRuntimeSlotActorTable     All 6 actor slots (0xC5C dwords)
0x3270  24      gRaceSlotStateTable        Slot state (6 x 4 bytes)
0x3288  ...     Masters state vars         DAT_0048F30C..0048F332
------
TOTAL:  0x32A6 (12966) bytes
```

The CRC-32 is computed over all 12966 bytes using a standard CRC-32/ISO-HDLC table at
`0x475160` (init = `0xFFFFFFFF`, final XOR = `0xFFFFFFFF`), then stored at offset +0x0C.

### `RestoreRaceStatusSnapshot` (0x4112C0)

Reverses serialization:

1. If `DAT_00494BBC == 0` (no data), returns 0.
2. Recomputes CRC-32 over the buffer.
3. If `stored_crc != computed_crc`, returns 0 (rejection).
4. Restores all fields from the buffer to their globals.
5. Special case: if decoded `g_selectedGameType == 0xFF`, remaps to `-1` (no active cup).
6. Also restores `DAT_0048F364/368/370/378` from within the schedule region (offsets
   +0x28, +0x40, +0x58, +0x70 relative to the schedule copy).
7. Returns 1 on success.

## 4. XOR Encryption Scheme

Both save files use the same algorithm with different keys:

```c
void xor_encrypt(byte *buf, int size, char *key) {
    int key_len = strlen(key);
    int ki = 0;
    for (int i = 0; i < size; i++) {
        buf[i] = buf[i] ^ key[ki] ^ 0x80;
        ki = (ki + 1) % key_len;
    }
}
// Decryption is identical (self-inverse).
```

| File | Key String | Key Length | VA |
|------|-----------|------------|-----|
| Config.td5 | `"Outta Mah Face!! "` | 18 | 0x463F9C |
| CupData.td5 | `"Steve Snake says : No Cheating! "` | 31 | 0x464084 |

The `^ 0x80` ensures that zero bytes don't directly reveal the key pattern.

## 5. CRC-32 Checksum Algorithm

Standard CRC-32 with precomputed 256-entry lookup table at `0x475160`:

```c
uint32_t crc32(byte *data, int length) {
    uint32_t crc = 0xFFFFFFFF;
    for (int i = 0; i < length; i++) {
        crc = (crc >> 8) ^ crc_table[(data[i] ^ crc) & 0xFF];
    }
    return ~crc;
}
```

First 4 entries of the table: `0x00000000, 0x77073096, 0xEE0E612C, 0x990951BA` --
confirming this is the standard ISO 3309 / ITU-T V.42 polynomial (0xEDB88320 reflected).

### Config.td5 CRC Placement

- CRC occupies bytes [0..3] of the buffer (file offset 0).
- During computation, bytes [0..3] are set to `0x10, 0x00, 0x00, 0x00` (placeholder).
- After computation, the result overwrites the placeholder.
- On load, the stored CRC is saved, the placeholder is restored, CRC is recomputed, and
  the two values are compared.

### CupData.td5 CRC Placement

- CRC occupies bytes [0x0C..0x0F] within the snapshot buffer (offset +12).
- Computed over the entire 0x32A6 bytes of the buffer.
- `ValidateCupDataChecksum` (0x411630) re-reads the file, decrypts, recomputes CRC-32
  over the full buffer, and compares against a passed-in expected value (param_3).

## 6. Save File Operations

### `WriteCupData` (0x4114F0)

```
1. Open "CupData.td5" for writing (mode "wb")
2. Allocate stack buffer of DAT_00494BBC bytes
3. XOR-encrypt: stack[i] = snapshot[i] ^ key[ki] ^ 0x80
4. fwrite(stack, 1, DAT_00494BBC, file)
5. fclose, return success if bytes_written == expected
```

### `LoadContinueCupData` (0x411590)

```
1. Open "CupData.td5" for reading (mode "rb")
2. fread into stack buffer (max 0x4000 bytes)
3. fclose
4. XOR-decrypt: snapshot[i] = stack[i] ^ key[ki] ^ 0x80
5. Store byte count in DAT_00494BBC
6. Call RestoreRaceStatusSnapshot() -- validates CRC internally
```

### `ValidateCupDataChecksum` (0x411630)

Called from `RaceTypeCategoryMenuStateMachine` to check if a valid cup save exists
**without** restoring it. Used to enable/disable the "Continue Cup" button.

```
1. Open "CupData.td5" for reading
2. fread + decrypt
3. Compute CRC-32 over decrypted buffer
4. Return (param_3 == computed_crc)
```

### `WritePackedConfigTd5` (0x40F8D0)

Serializes all persistent settings into buffer at `0x48F384` (5351 bytes total):

- Game options (laps, timers, traffic, cops, difficulty, dynamics, collisions)
- Controller bindings (P1 + P2 device descriptors + custom button maps)
- Audio settings (sound mode, SFX volume, music volume)
- Display settings (mode ordinal, fog, speed units, camera damping)
- Split-screen mode, catch-up assist
- Full NPC Racer Group Table (26 groups x 164 bytes = 4264 bytes of high scores)
- Car/track lock tables + max unlocked car index + all-cars flag
- Cheat activation flags (26 bytes)

CRC-32 at bytes [0..3], XOR-encrypted with "Outta Mah Face!! ".

### `LoadPackedConfigTd5` (0x40FB60)

Reads up to 0x1800 bytes, decrypts, validates CRC, restores all globals. Returns 0 on
failure (missing file or bad CRC), 1 on success.

**Load asymmetry:** Controller device descriptors are written from active slots
(`0x465660`, `0x465680`) but restored to backup slots (`0x4656A0`, `0x4656C0`). The active
slots are repopulated from hardware enumeration at runtime.

## 7. Cup Completion Unlocks

### `AwardCupCompletionUnlocks` (0x421DA0)

Called from `RunRaceResultsScreen` (state 15, button 4 = "Quit") after the final race
of a cup series.

**Preconditions (both must be true):**
1. `gRaceSlotStateTable.slot[0].companion_state_2 == 1` (player completed the race)
2. `gRuntimeSlotActorTable.slot[0].gap_0376[0xD] == 0` (player finished in 1st place)

**Unlock logic depends on `gNpcRacerGroupTable[DAT_004A2C90 * 0x29]` (group header byte):**

#### Header byte == 0 (Track groups: 0-3, 8-15)

Sets track completion flag:
```c
DAT_004A2C9C[DAT_004A2C90] = 1;
```
No car unlock. This records that the player has beaten this track's high-score group.

#### Header byte == 1 (Special groups: 4-7, 16-18)

Unlocks a specific car via hardcoded switch:

| NPC Group | Cup Context | Unlocked Car Index | Car Identity |
|-----------|-------------|-------------------|--------------|
| 4 | Era Race 1 | 0x15 (21) | Special 2 |
| 5 | Era Race 5 | 0x11 (17) | Chevelle SS |
| 6 | Era Race 3 | 0x18 (24) | A-Type Concept |
| 7 | Era Race 4 | 0x19 (25) | Nissan 240SX |
| 16 (0x10) | Era Race 2 | 0x17 (23) | Special 4 |
| 17 (0x11) | Era Race 6 | 0x1A (26) | Ford Mustang |
| 18 (0x12) | Bonus | 0x20 (32) | Cadillac Eldorado |

**Special case for group 18:** If `DAT_0048F364 == 0x20` (32 cars already at max), sets
`DAT_004962B0 = 1` (all-cars-unlocked flag) instead of unlocking car 0x20.

Unlock mechanics:
```c
DAT_00463E4C[car_index] = 0;   // clear lock flag
if (DAT_00463E0C < car_index + 1)
    DAT_00463E0C = car_index + 1;  // advance max accessible car count
```

### Cup Tier Gating

The cup selection menu (`RaceTypeCategoryMenuStateMachine`) uses `DAT_004962A8` to gate
access to higher cup tiers:

| DAT_004962A8 | Available Cups |
|-------------|----------------|
| 0 | Championship, Era only (Challenge/Pitbull/Masters/Ultimate grayed out) |
| 1 | + Challenge, Pitbull (Masters/Ultimate grayed out) |
| >= 2 | All cups available |

This value is persisted in Config.td5 at offset 0x148B (masked to 3 bits). The thunks at
0x410F60..0x4110A0 (called when a cup's sentinel race is reached) likely advance this tier.

## 8. "Continue Cup" End-to-End Flow

### Save (after each cup race)

```
RunRaceResultsScreen state 0x11:
  1. SerializeRaceStatusSnapshot()    // copies all cup state to buffer
  2. WriteCupData()                   // XOR-encrypts + writes CupData.td5
  3. Display "Block Saved OK" or "Failed to Save"
```

### Load (from race type menu)

```
RaceTypeCategoryMenuStateMachine state 0, button creation:
  1. ValidateCupDataChecksum(_, _, _)  // checks if CupData.td5 exists and is valid
  2. If valid:  create active "Continue Cup" button
     If invalid: create grayed-out "Continue Cup" preview button

State 3, button 2 ("Continue Cup") pressed:
  1. LoadContinueCupData()
     a. Read + decrypt CupData.td5
     b. RestoreRaceStatusSnapshot()
        - Validate CRC-32
        - Restore all cup globals: game type, race index, flags, schedule,
          results, actor state, slot state, masters permutation
  2. Set DAT_00497A6C = 0
  3. Navigate to screen 0x18 (RunRaceResultsScreen)
     - Results screen picks up at the restored race index
     - Player sees post-race screen for the last completed race
     - "Next Cup Race" button advances to the next race in the series
```

### Race Resumption

```
RunRaceResultsScreen state 13:
  1. Advance g_raceWithinSeriesIndex
  2. ConfigureGameTypeFlags()
     - If returns 0: cup is finished (sentinel 99 reached)
       -> Show "Cup Complete" buttons, call AwardCupCompletionUnlocks on quit
     - If returns 1: more races remain
       -> Show "Next Cup Race" button
  3. Player presses "Next Cup Race"
     -> State 15, button 0: sets DAT_004A2C8C = 2, transitions to car select
     -> After car select: begins next race with restored series state
```

### State Diagram

```
[Race Type Menu]
    |
    |-- "Cup Race" --> [Cup Type Select] --> [Car Select] --> [Race]
    |                                                           |
    |                                                     [Results Screen]
    |                                                      /          \
    |                                              "Save"           "Next Race"
    |                                                |                  |
    |                                          WriteCupData()     ConfigureGameTypeFlags()
    |                                                |                  |
    |                                           [CupData.td5]    if sentinel -> [Cup Complete]
    |                                                                  |
    |-- "Continue Cup" --> LoadContinueCupData() --> [Results Screen] --+
```

## 10. DrawRaceDataSummaryPanel (0x421E90)

Renders the race data summary panel in the post-race results screen. Called with a
racer slot index parameter to display that racer's statistics.

**Panel contents by game type:**

- **Cup types (1-6):** Track name, finish time (`MM:SS.cc`), speed (MPH or KPH based on
  `DAT_00466028`), final position, best lap time, and per-lap split times (up to 7 laps
  displayed, y-coords 0xC0-0x150).

- **Time Trial (7):** Track name, extended time format (`MM:SS.mmm`), speed readout.

- **Cop Chase (8):** Minimal display -- just damage/wanted statistics.

- **Drag Race (9):** Finish time with per-lap splits.

The panel is drawn to a secondary surface at `DAT_0049628C` (0x188 x 0x80 pixels) with
a black color-fill background. Speed unit is toggled by `DAT_00466028` (0=MPH format
string at `0x4660D8`, 1=KPH format string at `0x4660D0`).
