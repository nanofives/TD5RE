/fix Fix four camera issues; #1 and #4 likely share a root (fixed per-frame damping not scaled by delta-time):

1. COUNTDOWN SHAKE + PAUSE-AT-START ERRATIC. UpdateChaseCamera is gated on !g_td5.paused (~td5_camera.c:2678-2680); the countdown holds g_td5.paused while it runs (tick_race_countdown td5_game.c ~4487-4566) and flips it at timer==0, so the camera's spring state is stale on unfreeze and jumps. Car velocity ≈ 0 during the countdown also feeds garbage into the position interpolation (~1477-1484). Seed/settle the camera spring at race start and keep it smoothly updated during the countdown so the first racing frame doesn't snap; make pausing at the start moment not corrupt the spring state.

2. HIGH-MS JITTER. The radius/height smoothing uses a FIXED per-frame lerp (g_dampWeight=0.125f ~145; ~1038-1045) with no dt scaling, so at low FPS / high MS the smoothing oscillates. Make the damping frame-rate independent (scale the lerp by frame delta, or run it on the fixed sim tick) so it behaves the same at 30 and 60 fps.

3. INVISIBLE-CAR CAMERA GROUND CLIP. The special/invisible-car mode (gate on actor+0x379 ~724-735) uses the same height smoothing (~1043-1045) with NO ground clamp (a naive clamp was tried before and caused worse bugs — see comment ~1338-1343), so on slopes it dips below terrain. Add a careful ground-height floor for THIS mode only (clamp camera Y to a terrain-probe minimum + small offset) without reintroducing the old slope artifacts.
