# Branch Unification Plan — connected branch geometry for all tracks (TD5 + TD6)

> **STATUS 2026-06-17 — SUPERSEDED (geometry premise was wrong).** The "branches are
> world-displaced" assumption this plan is built on turned out to be a MEASUREMENT
> ARTIFACT. Raw coords (`re/tools/connect_branches.py characterize`) prove branch
> corridors are already geometrically CONNECTED to their forks (entry gap 0, rejoin
> within one span) on London (TD6) AND native Newcastle. No relocation/converter work
> is needed. The actual fix was the OPPOSITE: route TD6 through the native faithful path
> and suppress the displacement band-aids — `TD5RE_TD6_FAITHFUL` / `td5_track_td6_faithful()`,
> default ON (USER-CONFIRMED London fwd+rev + Egypt). The only still-useful part of this
> doc is the **band-aid site inventory in §3/Phase 3**, which guides the deletion of the
> now-dead code. The converter-relocation design (§2, Phase 1) is moot.


Goal: make branches geometrically part of the main road for EVERY track (native TD5
and converted TD6, forward and reverse), eliminate all "displaced branch" runtime
logic, and simplify adding new tracks. Plan produced 2026-06-17 (worktree TD5RE-london,
branch td6-london-props).

## Three findings that shape everything

1. **The converter does NOT touch geometry.** `re/tools/convert_td6_tracks.py` passes
   strip.dat/stripb.dat through verbatim except `remap_surface()` and
   `remap_type11_links()` (byte edits at span +0x01/+0x02 and link_prev). Span origins
   and vertices are untouched (`:1227-1242`). The displacement is inherent in the source
   TD6 data and faithfully preserved into the asset.

2. **CRITICAL CONSTRAINT — visible road is positioned by MODELS.DAT, not the strip
   origin (converted tracks).** Span→mesh is world-space nearest-center matching
   (`rebuild_span_display_list_mapping`, td5_track.c:1523-1573); MODELS.DAT meshes carry
   absolute baked world coords (`convert_td6_tracks._pack_mesh`). So relocating a branch
   span's collision/AI corridor WITHOUT also moving its visible road mesh desyncs render
   from the drivable surface AND scrambles the nearest-center map. Only the synthetic
   fallback quad uses `sp->origin`. **This forces relocation OFFLINE (move strip span +
   its mesh under one transform), not a runtime C pass.**

3. **SPLINE1.TD6 is route-only, not geometry.** SPLINE entries are `[laneA,flagA,laneB,
   flagB]` lateral fractions (0..255) of the racing line per main span — NOT XZ curves.
   They define no connector geometry. So connector spans must be GENERATED by
   interpolation; no connector-curve RE is required. (Already parsed by
   `apply_td6_spline_routes.py`; stays a route input only.)

Audit contradiction resolved: the `:2543` "gap=0 across all 56 London junctions" comment
is the old ORIGIN-only misread; the physically-relevant **road-EDGE world gap is ~8000u**
(my `td6_branch_audit`: London 56 forks displaced 5232–13250u, Newcastle native 4 forks
~3.0–3.6k). Edge gap is what the car/physics/render see, so it is the number relocation
must close. Update the stale comment.

## Chosen design: OFFLINE pre-bake into strip.json + models, one path for all tracks

New tool `re/tools/connect_branches.py` rewrites `strip.json`/`stripb.json` AND the branch
road meshes in `models.bin`/MODELS.DAT, per corridor:

1. **Entry fit (rotate+translate).** Rigid 2D transform T_entry mapping the corridor's
   entry edge (branch span `branch_lo` rail) onto the fork F's opening edge → entry seam
   gap→0, tangent-continuous. Apply T_entry to every branch span origin+vertices AND the
   corridor's road mesh group (same θ,t).
2. **Rejoin residual → generated connector.** After T_entry the far end (`branch_hi`)
   lands offset from rejoin span R (the two-ends problem — one rigid transform can't close
   both). Synthesize N short type-1 connector spans (interpolated rail/heading ribbon, N
   chosen so per-span heading delta stays under the smooth taper limit) + a generated road
   mesh; wire link_next/link_prev so F → branch_lo … branch_hi → connector → R is one
   continuous chain.
3. **Reclassify junctions.** Convert F/R type 8/11 → through-road type-1 (reuse
   `fix_td6_junction_zones.py:36-37`). Branch becomes ordinary main-ring road.

Editable-source layer (`td5_assetsrc.c:440` `td5_src_encode_strip`) is the bake target:
strip.json round-trips byte-exact to STRIP.DAT; TD6 8-u32 header tail + jump table +
surface grid live in `pre_span_hex` (keep verbatim). `strip_tool.py` is the round-trip
authority; `regen_td6_clean_strip.py` is the native→strip.json precedent. **Native TD5
tracks get a one-time pre-bake to loose strip.json** (same path London already uses).

Dependent tables after relocation:
- Jump table / lap remap: keep as PROGRESS map first cut (geometry ignores it; lap/
  checkpoint/minimap unchanged). Optional later renumber to one monotonic ring drops it +
  `branch_to_main_span` + minimap/HUD remaps.
- Surface grid (header[6]) + lighting zones: span/row-indexed, unaffected by relocation as
  long as span ordering + pre_span_hex preserved; append a row for connector spans (or
  point at an existing road row).
- Routes LEFT/RIGHT.TRK (+ reverse LEFTB/RIGHTB): regenerate via build_routes /
  apply_td6_spline_routes after the bake.
- Reverse STRIPB: same relocator (fork/rejoin roles swap, math identical) → lets us drop
  all `reverse_direction` gates.

## Effort phases (each builds via build_standalone.bat; headless validate via
## --AutoRace=1 --SkipIntro=1; success metric = every fork `td6_branch_audit` dist < ~1500)

- **Phase 0** — reconcile audit (done: edge gap is real); extend audit to log per-corridor
  REJOIN residual too; run on all branched tracks (TD6 levels 7,8,9,10,11,12,18,19,20,21,22
  per fix_td6_type11_links; native Newcastle/Maui/Courmayeur/HouseOfBez/MontegoBay). Output
  CSV: entry gap, rejoin residual, corridor span count, lane counts. Flag sharp-angle /
  huge-residual corridors (long connectors).
- **Phase 1** — build `connect_branches.py` (no engine change). Reuse strip_tool round-trip,
  build_routes, fix_td6_junction_zones flatten. Validate on the BAKED asset: audit gaps +
  residuals < ~1500; type-8/11 count → 0; engine loads (no placeholder fallback).
- **Phase 2** — race baked London with band-aids STILL ON (safety net): SUMMARY displaced=0,
  TELEPORT_DETECT silent over a full lap + deliberate branch entry, fwd + reverse; lap/
  checkpoint correct.
- **Phase 3 (payoff)** — delete band-aids file-by-file, rebuild+race after each:
  - td5_track.c: 4 force-main blocks in resolve_neighbor (:2621,:2674,:2735,:2785) → faithful
    branch follow; branch-return snap (:2840-2868); reverse main-ring guard (:2877-2884);
    synthetic boundary wall (:712-719,:801-804) + WALL_PEN reject/cap/deadzone/wallcap
    (:630-662,:775-785,:827-876) → original single reject; td6_flatten_junctions
    (:2116-2148); self-heal blacklist (:6560-6601, s_walker_is_traffic);
    routenet/branches_enabled knobs (:2547-2572).
  - td5_physics.c: S18 chassis-span reject + reverse-seam clause (:10103-10167).
  - td5_ai.c: td5_ai_branch_corridor_drivable TD6 gate (:4259), td5_ai_td6_drivable_band /
    td5_track_td6_road_band, traffic idle-heal/blacklist hooks.
  - td5_render.c: branch cull-center fallback (:2906-2962). td5_camera.c / td5_hud.c branch/
    ring-wrap altitude guards + minimap branch-fix become no-ops.
- **Phase 4 (optional, deepest)** — renumber to a single monotonic ring; drop jump table +
  branch_to_main_span + lap/minimap remaps. Highest risk to lap math; behind a per-track
  flag; do last.

RE still required: essentially none for geometry (splines are route-only; connectors
generated). Only open item: surface-grid row-append format if connectors get their own rows
(else reuse an existing road row).

Risks: sharp-angle forks where a single rigid entry fit leaves a large rejoin residual → the
generated connector may visibly cut a corner; the audit catches gaps but not connector
SMOOTHNESS → needs an eyeball pass. London's 5–13k displacements + the proven two-ends split
(corridor 2137–2148: entry (-8235,1231), rejoin (-2175,7077)) are exactly these.

## "Simpler new tracks" payoff

Today (per TD6 track): convert_td6_tracks → restore_td6_surface → fix_td6_type11_links →
fix_td6_junction_zones → apply_td6_spline_routes → tune TD5RE_TD6_* env knobs → seed
TD5RE_TD6_BRANCH_BLACKLIST. After: convert_td6_tracks → **connect_branches** → done; the
asset is a faithful single-network strip the unmodified engine drives — no env knobs, no
per-track blacklist. Native tracks gain the same one-shot pre-bake. Hundreds of
`g_active_td6_level`-gated lines across 6 files stop being a per-track maintenance tax.
